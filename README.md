# MiniMax Media Plugin

A single MCP plugin that lets Codex, Claude Code, and **Antigravity** all
drive the official MiniMax media APIs. One `git clone`, three AI clients.

![Sample image](assets/sample-panda.jpg)

> Generated by `text_to_image`, prompt *"a panda astronaut floating in pastel nebula"*.

## What it does

| Tool | What it produces |
| --- | --- |
| `text_to_image` | a still image (1024 × 1024 by default) |
| `generate_video` | a 6/10 s clip, optionally from a first frame |
| `query_video_generation` | poll an async video task |
| `text_to_audio` | TTS with a chosen voice |
| `list_voices` | enumerate system + cloning voices |
| `voice_clone` | clone a voice from an audio sample |
| `voice_design` | invent a voice from a description |
| `music_generation` | music from a prompt + lyrics |
| `play_audio` | play a generated audio file locally |

The plugin delegates everything to the official
[MiniMax-AI/MiniMax-MCP](https://github.com/MiniMax-AI/MiniMax-MCP) server
(MIT, [`minimax-mcp` on PyPI](https://pypi.org/project/minimax-mcp/)).
Nothing in this repo makes network calls of its own.

## Repo layout

```

assets/                            icons + a sample still image
```

## 1. Prerequisites

You need:

1. **Python ≥ 3.10** (the MCP server uses `FastMCP`).
2. **Astral's [`uv`](https://docs.astral.sh/uv/)**, which provides `uvx`.
   Install with `curl -LsSf https://astral.sh/uv/install.sh | sh` and confirm:

   ```bash
   which uvx          # e.g. /Users/you/.local/bin/uvx
   ```

3. **A MiniMax plan key** with image / video / voice / music included.
   Mainland sign-up at [platform.minimaxi.com](https://platform.minimaxi.com),
   global at [platform.minimax.io](https://platform.minimax.io).

## 2. Configure environment

Put these in your shell profile (`~/.zshrc`, `~/.bashrc`, `~/.config/fish/...`):

```bash
export MINIMAX_API_KEY="sk-cp-..."     # whatever your dashboard calls it
export MINIMAX_API_HOST="https://api.minimaxi.com"   # mainland; use api.minimax.io for global
export MINIMAX_MCP_BASE_PATH="$HOME/Desktop/minimax-output"
export MINIMAX_API_RESOURCE_MODE="url" # or "local" for disk writes
# only if your AI host can't find uvx on its own:
# export MINIMAX_UVX_PATH="/Users/you/.local/bin/uvx"
chmod 600 ~/.zshrc
```

Then `source ~/.zshrc`.

## 3. Install per host

### Codex desktop

> Settings → Plugins → "Add local plugin" → pick this repo's directory.

Codex reads `.codex-plugin/plugin.json` and spawns `uvx minimax-mcp` per
`.mcp.json`. Once loaded, ask Codex *"list voices"* to confirm it can see the
nine tools.

### Claude Code

```bash
# Add the repo as a marketplace and install the plugin
claude plugin marketplace add /absolute/path/to/minimax-plugin
claude plugin install minimax --from /absolute/path/to/minimax-plugin
```

Claude Code reads `.claude-plugin/plugin.json` and `.claude-plugin/marketplace.json`.
The `.mcp.json` at repo root is auto-picked up.

### Antigravity

Drop the repo at `~/.agents/plugins/minimax/` (symlinks are fine):

```bash
mkdir -p ~/.agents/plugins
ln -s "$(pwd)" ~/.agents/plugins/minimax
```

Antigravity shares Claude Code's plugin loader, so the same files work.

## 4. Verify

The skill description in `skills/minimax/SKILL.md` is loaded by all three

## 5. Quick smoke test (raw stdio)

For a non-host sanity check, drive the MCP server directly:

```python
import json, subprocess, os, sys

ENV = os.environ.copy()
proc = subprocess.Popen(
    ["uvx", "minimax-mcp"],
    stdin=subprocess.PIPE, stdout=subprocess.PIPE, stderr=subprocess.PIPE,
    env=ENV,
)
def rpc(method, params=None, _id=[0]):
    _id[0] += 1
    proc.stdin.write((json.dumps({"jsonrpc":"2.0","id":_id[0],"method":method,"params":params or {}})+"\n").encode())
    proc.stdin.flush()
    return json.loads(proc.stdout.readline())

print(rpc("initialize", {
    "protocolVersion": "2024-11-05",
    "capabilities": {},
    "clientInfo": {"name": "smoke", "version": "0"},
})["result"]["serverInfo"])
proc.stdin.write((json.dumps({"jsonrpc":"2.0","method":"notifications/initialized"})+"\n").encode())
proc.stdin.flush()
print([t["name"] for t in rpc("tools/list")["result"]["tools"]])
proc.terminate()
```

## Troubleshooting

- **`spawn uvx ENOENT`** — `uvx` is not on the AI host's `$PATH`. Run
  `scripts/uvx-resolver.sh` and either set `MINIMAX_UVX_PATH` or paste the
  absolute `command` into `.mcp.json`.
- **`API Error: invalid api key`** — verify the host you signed up against:
  mainland signups use `https://api.minimaxi.com`, global signups use
  `https://api.minimax.io`.
- **`voice clone user forbidden, should complete real-name verification`** —
  voice cloning on the mainland plan requires 实名认证. Visit
  `https://platform.minimaxi.com/user-center/basic-information` to finish.
- **Files vanish after a few minutes** — you are running with
  `RESOURCE_MODE=url`. Either set `RESOURCE_MODE=local` or `curl -L` the
  signed URL into your own outputs directory.
- **Sandbox/permission errors writing the key** — store the key outside any
  git-tracked file. The cleanest is `~/.zshrc` (`chmod 600`) like above; do
  not paste it into `.mcp.json`.
